# Independently verified module evidence

Each JSON file in this directory is generated by
`tests/verify-accepted-evidence-artifact.py` from its linked immutable GitHub
Actions aggregate artifact. The report binds one `family/module` key to the
tested commit, aggregate-index digest, and GitHub-published ZIP digest.

The directory deliberately contains both full-catalog reports and focused
refreshes. For example, the full catalog artifact from
[`29652229194`](https://github.com/Yunushan/linux-software-installer/actions/runs/29652229194)
verified 371 indexed cells, while a focused run may supersede one report after
that module's contract changes. Always use the individual report's `run_url`,
`commit_sha`, `index_sha256`, and `artifact_sha256`; do not infer those values
from another report in this directory.

These reports are verification inputs, **not** admissions: every legacy row
still requires an exact parity review, and service contracts also require
accepted disposable-VM/systemd attestations. Do not add a row to
`docs/accepted-evidence.tsv` or change its inventory disposition solely because
a report exists here.
